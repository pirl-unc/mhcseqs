mhcseqs

Self-contained pipeline for downloading, curating, and extracting binding grooves from MHC protein sequences.

Install CLI Python API Output Algorithm Species Modules

Installation

pip install mhcseqs

For development:

git clone https://github.com/openvax/mhcseqs.git
cd mhcseqs
./develop.sh    # pip install -e ".[dev]"
./test.sh       # pytest
./lint.sh       # ruff

Requirements: Python 3.10+, mhcgnomes (installed automatically).

CLI Reference

mhcseqs build

Downloads FASTA sources (IMGT/HLA + IPD-MHC), parses alleles, selects two-field representatives, extracts binding grooves, and runs validation.

mhcseqs build [--output-dir DIR] [--data-dir DIR]
OptionDescription
--output-dirDirectory for output CSVs (default: repo root)
--data-dirDirectory for downloaded FASTA files (default: data/fasta/)

Produces five files:

FileDescription
mhc-seqs-raw.csvEvery parsed protein entry from both databases
mhc-full-seqs.csvOne representative mature protein per two-field allele group
mhc-binding-grooves.csvStructural decomposition: groove + Ig domain + tail
mhc-merge-report.txtDetails on allele group merging decisions
mhc-validation-report.txtSanity check results

mhcseqs lookup

Query built CSVs by allele name.

mhcseqs lookup "HLA-A*02:01"
mhcseqs lookup "Mamu-A1*001:01"

Python API

Allele lookup

import mhcseqs

# Build the database (downloads FASTA sources, only needed once)
mhcseqs.build()

# Look up any allele → AlleleRecord with everything
r = mhcseqs.lookup("HLA-A*02:01")
r.allele             # "HLA-A*02:01" (two-field)
r.full_allele        # "HLA-A*02:01:01:100" (full resolution)
r.sequence           # full protein (with signal peptide)
r.mature_sequence    # signal peptide removed (computed property)
r.mature_start       # signal peptide length (24)
r.groove1            # α1 domain (90 aa)
r.groove2            # α2 domain (93 aa)
r.ig_domain          # α3 Ig-fold (95 aa)
r.tail               # TM + cytoplasmic
r.species_category   # "human"

# Apply mutations (IEDB-style, e.g. "K66A")
m = mhcseqs.lookup("HLA-A*02:01", mutations=["K66A", "D77S"])

Groove extraction algorithm

The parser is alignment-free — it uses conserved Cys-Cys disulfide pairs in Ig-fold domains as structural landmarks.

Domain architecture

CLASS I alpha chain:

  raw pos:  0       SP   ms        ms+90  C1  C1+10     C2+20     s_C2+20
            |--------|    |          |     |    |          |         |
            [ signal ]    [-- groove1 --]--[--- groove2 ---]--[-- ig --]--[ tail ]
            [ peptide]    [     α1      ]  [      α2       ]  [  α3   ]
                          |<-- 90 aa -->|  | C1.......C2   |  |C1..C2 |
                                           |<--- 93 aa --->|

CLASS II alpha chain:

  raw pos:  0       SP   ms           ms+83  C1         C2
            |--------|    |              |    |          |
            [ signal ]    [-- groove1 ---]----[--- ig ------]--[ tail ]
            [ peptide]    [     α1       ]    [    α2       ]

CLASS II beta chain:

  raw pos:  0       SP   ms  C1b       C2b      C1i         C2i
            |--------|    |   |          |        |           |
            [ signal ]    [------- groove2 ------]--[-- ig ----]--[ tail ]
            [ peptide]    [         β1           ]  [   β2     ]

How it works

  1. Find all Cys-Cys pairs with 48–72 residue separation (Ig-fold signature).
  2. Select the pair in the expected positional range, preferring the one whose separation is closest to the domain's canonical value.
  3. Infer signal peptide length by subtracting the known mature-position constant from the raw Cys position.
  4. Slice groove, Ig domain, and tail using offsets relative to the anchor pair.

Groove status codes

StatusMeaning
okPrimary anchor found, groove extracted normally
alpha3_fallbackClass I: used α3 Cys pair instead of α2
beta1_only_fallbackClass II β: only β1 Cys pair available
fragment_fallbackShort sequence treated as raw groove fragment
not_applicableB2M, MICA, MICB, HFE (not peptide-binding)
too_shortSequence < 70 aa
no_cys_pairsNo valid Cys-Cys pairs found
no_alpha2_pairClass I: no Cys pair in α2 range

Module reference

mhcseqs.groove

Core groove extraction. Key functions:

  • extract_groove(seq, mhc_class, ...) — main dispatcher
  • parse_class_i(seq, ...) — class I alpha chain parser
  • parse_class_ii_alpha(seq, ...) — class II alpha chain parser
  • parse_class_ii_beta(seq, ...) — class II beta chain parser
  • find_cys_pairs(seq) — locate Ig-fold Cys-Cys pairs
  • AlleleRecord — dataclass with all parsed fields

mhcseqs.alleles

Allele name parsing via mhcgnomes:

  • parse_allele_name(name) — parse with mhcgnomes
  • normalize_allele_name(name) — canonical two-field form
  • infer_gene(allele) — extract gene (e.g. "A", "DRB1")
  • infer_mhc_class(allele) — "I" or "II"

mhcseqs.species

Species normalization:

  • normalize_species(raw) — 29-class fine-grained
  • normalize_mhc_species(raw) — 7-class category
  • get_latin_name(raw) — canonical Latin name
  • get_canonical_prefix(raw) — MHC naming prefix
  • LATIN_NAMES, CANONICAL_MHC_PREFIXES — lookup dicts

mhcseqs.pipeline

Three-step build pipeline:

  • build_raw_index(fasta_paths, out_csv) — Step 1: parse FASTA → raw CSV
  • build_full_seqs(raw_csv, out_csv) — Step 2: select representatives
  • build_binding_grooves(full_csv, out_csv) — Step 3: extract grooves

mhcseqs.validate

Post-build sanity checks:

  • validate_build(raw_csv, full_csv, groove_csv) — run all checks
  • format_validation_report(warnings, stats) — human-readable report

mhcseqs.download

FASTA source downloading:

  • download_all(dest_dir) — download IMGT/HLA + IPD-MHC
  • SOURCES — dict of source URLs and metadata
